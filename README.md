# ВГору AI — Інтелектуальний помічник для підбору гірських маршрутів

## Elevator Pitch

Туристи, які планують похід у Карпати, стикаються з однією і тією ж проблемою: на сайті десятки маршрутів, і щоб обрати підходящий, потрібно самостійно читати описи, порівнювати складність, висоту, тривалість і поради щодо безпеки. Це забирає багато часу і легко заплутатися.

**ВГору AI** вирішує цю проблему за один запит. Користувач описує свої побажання у довільній формі — «легкий маршрут для новачка з дитиною на один день» або «хочу панораму, не більше 10 км» — і отримує персоналізовану рекомендацію з обґрунтуванням, конкретними характеристиками маршруту та практичними порадами щодо спорядження і безпеки. Система не вигадує відповіді — вона шукає у власній базі знань і формує відповідь на основі реальних даних.

---

## Цільова аудиторія

- **Туристи-початківці**, які вперше плануть похід у Карпати і не знають, з чого почати
- **Сім'ї з дітьми**, яким важливо знайти безпечний і посильний маршрут
- **Люди, що планують одноденну вилазку** і хочуть швидко підібрати варіант без довгого читання описів
- **Досвідчені туристи**, які шукають нові маршрути за конкретними критеріями (висота, регіон, складність)
- **Відвідувачі сайту VGoru**, яким потрібна персоналізована консультація замість самостійного перегляду каталогу

## Архітектура та ШІ-паттерни

### RAG (Retrieval-Augmented Generation)

Модуль використовує власну базу знань — файл `knowledge_base.txt`, який містить структуровані описи 15 маршрутів Українських Карпат із полями: назва, регіон, висота, складність, відстань, тривалість, детальний опис, поради та теги. Окремо зберігається загальна інформація про безпеку, спорядження та сезонність.

При кожному запиті система парсить цей файл і передає релевантні дані моделі як контекст. Модель не вигадує маршрути — вона отримує їх із документа. Це класичний RAG без векторної бази даних.

### Function Calling (Агентний підхід)

`service.py` реалізує агентний цикл: модель Gemini самостійно вирішує, яку функцію викликати, і може робити це кілька разів поспіль до формування фінальної відповіді. Доступні функції:

| Функція | Призначення |
|---|---|
| `search_routes` | Повнотекстовий пошук за назвою, описом, регіоном або тегами |
| `filter_routes` | Фільтрація за складністю, тривалістю, відстанню, регіоном та тегами |
| `get_route_details` | Повна інформація про конкретний маршрут із порадами |
| `get_general_info` | Загальна інформація про безпеку, спорядження та сезонність |

### Структура модуля

```
ai_assistant/
├── __init__.py
├── apps.py                  # Django AppConfig
├── urls.py                  # URL-маршрути: /ai/, /ai/api/chat/, /ai/api/clear/
├── views.py                 # HTTP-обробники, збереження історії в сесії
├── service.py               # Агентний цикл (Gemini API + Function Calling)
├── tools.py                 # Визначення функцій та логіка їх виконання
├── knowledge_base.txt       # База знань маршрутів (plain text, RAG)
└── templates/
    └── ai_assistant/
        └── chat.html        # Веб-інтерфейс чату
```

### Управління станом

Історія розмови зберігається у Django-сесії (серверна сторона, без додаткової бази даних). Максимум 10 пар повідомлень — старіші автоматично обрізаються, щоб не перевищувати ліміт токенів. Кнопка «Нова розмова» очищає сесію через окремий API-ендпоінт.

## Встановлення та запуск

### 1. Залежності

```bash
pip install google-genai python-dotenv
```

Або додайте до `requirements.txt`:

```
google-genai
python-dotenv
```

### 2. Структура файлів у проекті

Скопіюйте папку `ai_assistant/` у корінь проекту поруч із `manage.py`:

```
vgoru/
├── ai_assistant/        ← цей модуль
├── mountains_roads/
├── vgoru/
├── .env                 ← файл із ключем API
└── manage.py
```

### 3. Отримання API-ключа

Зареєструйтесь та отримайте безкоштовний ключ на [Google AI Studio](https://aistudio.google.com/apikey).

### 4. Налаштування .env

Створіть файл `.env` у корені проекту (поруч із `manage.py`):

```
GEMINI_API_KEY=AIza...ваш_ключ...
```

> ⚠️ Додайте `.env` до `.gitignore`, щоб ключ не потрапив у репозиторій:
> ```
> echo ".env" >> .gitignore
> ```

### 5. Підключення до Django-проекту

**`vgoru/settings.py`** — додати аппку та завантаження `.env`:

```python
from pathlib import Path
import os
from dotenv import load_dotenv

load_dotenv()

# ... решта settings без змін ...

INSTALLED_APPS = [
    # ... існуючі аппки ...
    'ai_assistant.apps.AiAssistantConfig',  # ← додати
]
```

**`vgoru/urls.py`** — підключити URL-маршрути:

```python
from django.urls import path, include

urlpatterns = [
    # ... існуючі маршрути ...
    path('ai/', include('ai_assistant.urls', namespace='ai_assistant')),  # ← додати
]
```

**`mountains_roads/templates/mountains_roads/base.html`** — додати посилання у navbar:

```html
<li><a href="{% url 'ai_assistant:chat' %}">ВГору AI</a></li>
```

### 6. Запуск

```bash
python manage.py runserver
```

ШІ-помічник доступний за адресою: **http://127.0.0.1:8000/ai/**

> Міграції не потрібні — модуль не має власних моделей бази даних.

## Оцінка вартості

Модель: `gemini-2.5-flash` (безкоштовний тариф — 15 запитів/хв, 1 500 запитів/день).

Один діалог із підбором маршруту в середньому займає 2–3 виклики API (агентний цикл) і витрачає приблизно 1 000–2 500 токенів. На безкоштовному тарифі це дозволяє обробити сотні запитів на день без додаткових витрат.
